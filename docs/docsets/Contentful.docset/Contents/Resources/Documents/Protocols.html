  • PersistenceIntegration

    Conform to this protocol and initialize your Client instance with the persistenceIntegration initialization parameter set to recieve messages about creation and deletion of Entrys and Assets in your space when doing sync operations using the Client.initialSync() and Client.nextSync()methods. Proper conformance to this protocol should enable persisting the state changes that happen in your Contentful space to a persistent store such asCoreData`.

  • ChainableQuery

    Protocol which enables concrete query implementations to be ‘chained’ together so that results can be filtered by more than one Query.Operation or other query. Protocol extensions give default implementation so that all concrete types, Query, AssetQuery, FilterQuery, and QueryOn<EntryType>, can use the same implementation.

    See more

    The base abstract type for querying Contentful entries. The contained operations in the default implementation of this protocol can only be used when querying against the `/entries/ endpoint of the Content Delivery and Content Preview APIs.
